# Quick Task: Extract Mappers from Controllers - SUMMARY

## One-Liner

Extracted mapping logic from PlayerController, TeamController, and ManagerController into dedicated, reusable @Component mapper classes to enforce Hexagonal Architecture separation of concerns.

## Context

The web controllers embedded mapping methods (toDto, mapToDto, toDomain) that violated the Single Responsibility Principle in Hexagonal Architecture. Controllers should handle HTTP request/response lifecycle only; transformation logic should be in separate adapters.

### Task 1-3: Mapper Creation

Created three new mapper classes in `src/main/java/br/com/saveeditor/brasfoot/adapters/in/web/mapper/`:

- **PlayerMapper.java**: Converts Player ↔ PlayerDto
- **TeamMapper.java**: Converts Team ↔ TeamDto  
- **ManagerMapper.java**: Converts Manager ↔ ManagerDto and ManagerUpdateRequest → Manager

Each mapper:
- Annotated with `@Component` for Spring auto-injection
- Provides `toDto()` for single object conversion
- Provides `toDtoList()` for collection conversion
- Provides `toDomain()` (ManagerMapper only) for request-to-domain conversion
- Includes Javadoc explaining responsibilities

**Verification**: `mvn clean compile` passed without errors.

### Task 4-6: Controller Refactoring

Updated all three controllers to use injected mappers:

**PlayerController**:
- Added `PlayerMapper playerMapper` field with constructor injection
- Replaced 3x `mapToDto()` calls with `playerMapper.toDto()`
- Replaced 2x inline stream mappings with `playerMapper.toDtoList()`
- Removed 21 lines (embedded mapToDto method)

**TeamController**:
- Added `TeamMapper teamMapper` field with constructor injection
- Replaced 3x `mapToDto()` calls with `teamMapper.toDto()`
- Replaced 2x inline stream mappings with `teamMapper.toDtoList()`
- Removed 18 lines (embedded mapToDto method)

**ManagerController**:
- Added `ManagerMapper managerMapper` field with constructor injection
- Replaced 4x mapping calls with `managerMapper.toDto()` or `managerMapper.toDomain()`
- Replaced 1x inline stream mapping with `managerMapper.toDtoList()`
- Removed 31 lines (embedded toDto and toDomain methods)
