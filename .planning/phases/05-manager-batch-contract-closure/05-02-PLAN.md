## Why This Matters

HTTP 207 Multi-Status is the correct response for partial-success operations (RFC 7231). When updating 5 managers and 2 fail:
- Current API returns 200 (false positive — client thinks all succeeded)
- Correct API returns 207 (client knows some failed)
- Error must reference array index (0, 1, 2...) so client can correlate with request

This is a breaking change to the batch API contract, but it's necessary for reliability and matches industry standards (e.g., Google Batch API, AWS batch operations).

## Architecture Context

- Spring Boot 3.2.1 supports HttpStatus.MULTI_STATUS via ResponseEntity
- All batch use cases (GetManagerUseCase, UpdateTeamUseCase, etc.) are service-based
- Domain models have validation that throws IllegalArgumentException on failure
- Reflection can fail silently if field names are wrong (no exception needed)
- Session state is in-memory; no database transactions

## Implementation Strategy

1. Create generic BatchResponse<T> and BatchResult<T> DTOs
2. Update ManagerManagementService.batchUpdateManagers() to wrap results with index and success flag
3. Update ManagerController.batchUpdateManagers() to return 207 when any item failed
4. Align Team/Player batch endpoints with same contract (same wave, different plan)
5. Update BatchUpdateManagerUseCase port to return BatchResponse instead of List

package br.com.saveeditor.brasfoot.adapters.in.web.record.out;

public class BatchResult<T> {
    private int index;
    private boolean success;
    private T data;
    private String error;

    public BatchResult(int index, boolean success, T data, String error) {
        this.index = index;
        this.success = success;
        this.data = data;
        this.error = error;
    }

    public BatchResult() {}

    public int getIndex() {
        return index;
    }

    public void setIndex(int index) {
        this.index = index;
    }

    public boolean isSuccess() {
        return success;
    }

    public void setSuccess(boolean success) {
        this.success = success;
    }

    public T getData() {
        return data;
    }

    public void setData(T data) {
        this.data = data;
    }

    public String getError() {
        return error;
    }

    public void setError(String error) {
        this.error = error;
    }

    // Convenience factory methods
    public static <T> BatchResult<T> success(int index, T data) {
        return new BatchResult<>(index, true, data, null);
    }

    public static <T> BatchResult<T> failure(int index, String error) {
        return new BatchResult<>(index, false, null, error);
    }
}
```

Generic type parameter T. Index field enables error mapping back to request array. Factory methods for concise construction.

@PatchMapping("/batch")
@Operation(summary = "Batch update managers", 
           description = "Updates multiple managers in a single request. Returns 207 Multi-Status if any updates fail.")
public ResponseEntity<BatchResponse<ManagerDto>> batchUpdateManagers(
        @PathVariable String sessionId,
        @RequestBody List<ManagerBatchUpdateCommand> commands) {
    
    BatchResponse<Manager> response = batchUpdateManagerUseCase.batchUpdateManagers(sessionId, commands);
    
    // Convert Manager results to ManagerDto
    List<BatchResult<ManagerDto>> dtoResults = response.getResults().stream()
        .map(result -> {
            if (result.isSuccess()) {
                return BatchResult.success(result.getIndex(), toDto(result.getData()));
            } else {
                return BatchResult.failure(result.getIndex(), result.getError());
            }
        })
        .collect(Collectors.toList());
    
    BatchResponse<ManagerDto> dtoResponse = new BatchResponse<>(dtoResults);
    
    // Determine status: 207 if any failed, 200 if all succeeded
    boolean anyFailed = response.getResults().stream().anyMatch(r -> !r.isSuccess());
    HttpStatus status = anyFailed ? HttpStatus.MULTI_STATUS : HttpStatus.OK;
    
    return ResponseEntity.status(status).body(dtoResponse);
}
```
